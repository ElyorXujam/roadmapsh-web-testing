## Submitting a 500+ character search query triggers a server error instead of input validation or truncation

## Summary
Submitting a 500+ character query in the search bar results in a server error rather than graceful input truncation or a user-friendly validation message.

## Environment
- Browser: Chrome 125.0
- Os: Windows 11
- Account Type: Registered User

## Steps to Reproduce
1. Go to `https://roadmap.sh`
2. Click the search bar to focus it
3. Paste a string of 500 or more characters (e.g., repeat "abcdefghij" 50 times)
4. Press Enter to submit the query
5. Observe the UI response and check DevTools → Network tab for any error response

## Expected Behavior
The search input should either enforce a character limit and block submission, or handle oversized input gracefully by truncating it and returning relevant results — with no server error.

## Actual Behavior
The server returns an error response. No user-facing message is shown to explain what went wrong or how to correct the input.
