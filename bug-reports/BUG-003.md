## System accepts deleted GitHub repository links without validation during project submission

## Summary
After a user submits a project with a GitHub repository link and then deletes that repository, re-submitting the same dead link is silently accepted with no error or validation.

## Environment
- Browser: Chrome 125.0
- Os: Windows 11
- Account Type: Registered User

## Steps to `https://roadmap.sh/frontend/projects` (a roadmap with a Projects section)
2. Submit a project using a valid GitHub repository URL
3. Confirm the submission is accepted and the link is displayed
4. On GitHub, navigate to that repository's settings and delete it
5. Return to the roadmap.sh and press a share button
6. Observe whether the platform validates the URL's existence

## Expected Behavior
The platform should validate the submitted GitHub URL and display an error such as "This repository could not be found. Please check the link or submit a different repository." Dead links should not be accepted or stored.

## Actual Behavior
The deleted repository URL is accepted without any validation check or error message. The stale link is stored and visible to other users, who will encounter a 404 on GitHub when they click it.
