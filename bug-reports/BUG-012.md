## Roadmap node state does not sync across multiple tabs without manual page refresh

## Summary
When a user marks a roadmap node as "Done" in one browser tab, the change does not appear in other simultaneously open tabs showing the same roadmap until the page is manually refreshed.

## Environment
- Browser: Chrome 125.0
- Os: Windows 11
- Account Type: Registered User

## Steps to Reproduce
1. Go to `https://roadmap.sh/roadmaps/frontend` in Tab 1
2. Open a new tab and go to the same URL in Tab 2
3. In Tab 1, right-click any node (e.g., "HTML") and select "Mark as Done"
4. Switch to Tab 2 without refreshing
5. Observe the state of the same node in Tab 2

## Expected Behavior
The node marked as "Done" in Tab 1 should also appear as "Done" in Tab 2 — either immediately via WebSocket push or within a reasonable polling interval — without requiring a manual page refresh.

## Actual Behavior
Tab 2 still shows the node in its previous state. The update only becomes visible after pressing F5 to manually refresh.
