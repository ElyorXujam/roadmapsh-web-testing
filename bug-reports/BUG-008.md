## System discards deep-link destination and redirects to default dashboard after post-login authentication

## Summary
When a logged-out user navigates to a protected URL and is redirected to log in, the original destination is discarded — after login they are taken to the default dashboard instead of the page they originally tried to access.

## Environment
- Browser: Chrome 125.0
- Os: Windows 11
- Account Type: Registered User (logged out at session start)

## Steps to Reproduce
1. Ensure you are fully logged out of roadmap.sh
2. Paste a protected URL directly into the address bar — e.g., `https://roadmap.sh/roadmaps/custom`
3. Note the URL you are redirected to
4. Complete the login process using valid credentials
5. Observe the page you land on after successful login

## Expected Behavior
After a successful login, the user should be redirected to the original protected URL they were trying to reach (e.g., `https://roadmap.sh/roadmaps/custom`).

## Actual Behavior
After login, the user is taken to the default dashboard or home page. The original destination URL is discarded and the user must navigate back manually.
