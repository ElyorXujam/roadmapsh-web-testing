## Roadmap node text renders too small and is illegible on mobile viewports due to lack of responsive scaling

## Summary
On a 390px mobile viewport, roadmap node text is rendered extremely small (estimated 6–8px) with no responsive adjustment, making the roadmap effectively unreadable without manual pinch-to-zoom.

## Environment
- Browser: Chrome 125.0 with DevTools mobile emulation (iPhone 12 Pro — 390px viewport)
- Os: Windows 11
- Account Type: Registered User

## Steps to Reproduce
1. Open Chrome DevTools (F12) and enable the Device Toolbar (Ctrl+Shift+M)
2. Select "iPhone 12 Pro" from the device dropdown (390px viewport width)
3. Go to `https://roadmap.sh/roadmaps/frontend`
4. Allow the page to fully load
5. Observe the roadmap canvas — specifically the text labels inside nodes
6. Attempt to read a node label without pinching to zoom

## Expected Behavior
The roadmap viewer should apply a responsive layout for small screens. Node text should be legible at a minimum rendered size of ~12px per WCAG guidelines, either via an adjusted initial zoom level or a simplified mobile layout.

## Actual Behavior
The roadmap canvas renders at desktop scale. Node text is approximately 6–8px — effectively unreadable — with no font scaling, zoom adjustment, or mobile-specific layout applied.
