## Escape key fails to close search bar or dismiss dropdown when search input is focused

## Summary
Pressing the Escape key while the search bar is open and focused does not close the search bar or dismiss any dropdown, leaving the user stuck in the search state.

## Environment
- Browser: Chrome 125.0
- Os: Windows 11
- Account Type: Registered User

## Steps to Reproduce
1. Go to `https://roadmap.sh`
2. Click the search bar to open and focus it
3. Optionally type a query to trigger any visible dropdown
4. Press the Escape key
5. Observe whether the search bar or dropdown closes

## Expected Behavior
Pressing Escape should close the search dropdown (if open) or dismiss the search bar entirely, returning the user to the normal browsing view.

## Actual Behavior
The Escape key has no effect. The search bar and any open dropdown remain visible and active.
