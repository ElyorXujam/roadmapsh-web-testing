## Roadmap nodes lack keyboard focus and interactivity, breaking accessibility for non-mouse users

## Summary
Roadmap nodes are not keyboard-navigable — users cannot Tab to nodes, open them with Enter, or mark their status via keyboard, making the roadmap inaccessible without a mouse.

## Environment
- Browser: Chrome 125.0
- Os: Windows 11
- Account Type: Registered User

## Steps to Reproduce
1. Go to `https://roadmap.sh/roadmaps/frontend`
2. Click anywhere on the page to ensure focus is within the document
3. Press Tab repeatedly to attempt navigating between roadmap nodes
4. Attempt to press Enter on any focused node to open its detail panel
5. Attempt to mark a node as "Done" using only the keyboard

## Expected Behavior
Roadmap nodes should be focusable via Tab, openable via Enter, and support keyboard-based status changes. Progress set via keyboard should be saved correctly, the same as mouse interactions.

## Actual Behavior
Keyboard navigation through nodes is not possible. Tab focus does not land on any roadmap node. There is no way to open a node or change its status without using a mouse.
