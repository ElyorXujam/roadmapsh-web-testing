## Country combobox lacks accessible label, causing screen readers to announce role without field context

## Summary
The country select combobox on the profile settings page has no accessible label, causing screen readers to announce only the selected value and role ("combobox") with no indication of what the field controls.

## Environment
- Browser: Chrome 125.0 with axe DevTools accessibility extension
- Os: Windows 11
- Account Type: Registered User

## Steps to Reproduce
1. Go to `https://roadmap.sh/account/update-profile`
2. Locate the country selection dropdown (Radix UI combobox)
3. Inspect the element in DevTools — confirm `role="combobox"` is present but no `aria-label` or `aria-labelledby` attribute exists
4. Use a screen reader (e.g., NVDA + Chrome) and Tab to this control
5. Observe what is announced when focus lands on the combobox

## Expected Behavior
The screen reader should announce the field's label and role — e.g., "Country, collapsed, select" — so the user understands what data this dropdown collects.

## Actual Behavior
The screen reader announces only the currently selected value (or nothing if empty) and the role "combobox", with no field label. Users cannot determine this is the "Country" field without visual context. Also confirmed by automated axe scan (Impact: Critical, Rule ID: `select-name`).
