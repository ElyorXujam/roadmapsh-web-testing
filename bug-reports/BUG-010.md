## Icon-only button lacks accessible label, causing screen readers to announce no descriptive purpose

## Summary
An icon-only button on the Frontend roadmap page has no accessible label (`aria-label`, `title`, or inner text), causing screen readers to announce only "button" with no indication of its purpose.

## Environment
- Browser: Chrome 125.0 with axe DevTools accessibility extension
- Os: Windows 11
- Account Type: Registered User

## Steps to Reproduce
1. Go to `https://roadmap.sh/roadmaps/frontend`
2. Inspect the button element with class `.rounded-r-full` using browser DevTools
3. Confirm the element has no inner text and no `aria-label`, `aria-labelledby`, or `title` attribute
4. Use a screen reader (e.g., NVDA + Chrome) and Tab to this button
5. Observe what the screen reader announces when the button receives focus

## Expected Behavior
The screen reader should announce a descriptive label for the button — e.g., "Submit search" — so that keyboard and assistive technology users understand the button's purpose.

## Actual Behavior
The screen reader announces only "button" with no name or purpose. A blind or low-vision user has no way to understand what this button does from keyboard navigation alone. Also confirmed by automated axe scan (Impact: Critical, Rule ID: `button-name`).
