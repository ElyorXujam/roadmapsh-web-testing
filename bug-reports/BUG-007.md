## Headline field on profile settings lacks character limit and allows oversized string submissions

## Summary
The "Headline" field on the profile settings page has no character limit, allowing an arbitrarily long string to be saved with no validation or feedback.

## Environment
- Browser: Chrome 125.0
- Os: Windows 11
- Account Type: Registered User

## Steps to Reproduce
1. Go to `https://roadmap.sh/account/update-profile`
2. Locate the "Headline" text input field
3. Paste a very long string — e.g., repeat a sentence 20 times to reach ~2,000+ characters
4. Observe whether any character counter, limit badge, or input restriction appears
5. Click "Save" and observe the result

## Expected Behavior
The field should enforce a reasonable character limit (e.g., 100–160 characters), display a live counter, and prevent saving a headline that exceeds the limit.

## Actual Behavior
The field accepts unlimited text with no counter, no warning, and no error on save. The oversized headline is stored without any validation feedback.
